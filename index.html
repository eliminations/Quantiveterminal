capabilities

Four phases.
Complete transformation.

01 Scan

Codebase Analysis

Recursive scan of files, directories, identifiers, strings, configs, and environment variables. Builds complete identity map.

  • Package metadata
  • Source code identifiers
  • UI strings & copy
  • Config & env vars
02 Refactor

Identity Transformation

Case-aware, context-sensitive replacement. Handles PascalCase, camelCase, kebab-case, snake_case, and SCREAMING_SNAKE.

  • Word boundary detection
  • Import path updates
  • File/dir renaming
  • Zero partial matches
03 Differentiate

UI/UX Transformation

Natural language style descriptions become concrete design tokens. Three scope levels for surgical to structural changes.

  • Light: colors, typography
  • Medium: + components
  • Heavy: + structure
  • Framework detection
04 Validate

Confidence Verification

Comprehensive checks ensure zero residual references, passing tests, successful builds, and complete visual differentiation.

  • Residual reference scan
  • Build verification
  • Test suite execution
  • Transformation report

ui_scope

Choose your depth.

Light

--ui-scope light

Visual tweaks only. Colors, typography, spacing, shadows. No structural changes to components.

+ Color palette
+ Typography scale
+ Spacing rhythm
+ Border radius
+ Shadow system
- Components
- Layouts
Recommended

Medium

--ui-scope medium

Component refactors and layout changes. Structural modifications that preserve functionality.

+ All light scope
+ Component structure
+ Layout grids
+ Navigation patterns
+ Card/container redesign
- Interactions
- Animations

Heavy

--ui-scope heavy

Complete structural overhaul. New interaction patterns, animations, and page architecture.

+ All medium scope
+ Interaction patterns
+ Animation system
+ Page structure
+ Component hierarchy
+ Motion choreography
+ Brand assets
